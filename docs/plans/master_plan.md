## 단계별 의존성 다이어그램

### 의존성 흐름
```
MVP Phase (Foundation)
    │
    ├─► 1인칭 컨트롤러 ───┬─► Alpha Phase (Network Layer)
    │                      │    │
    ├─► 고래 AI 기초 ──────┤    ├─► 네트워크 동기화 ───┬─► Beta Phase (Content)
    │                      │    │                        │
    ├─► 배 물리 시스템 ────┤    ├─► 역할 선택 시스템 ───┤    ├─► AI 고도화
    │                      │    │                        │    │
    └─► 페이즈 시스템 기초─┘    └─► 동기화된 게임플레이─┘    ├─► 체크포인트 저장
                                                           │
                                                           ├─► 환경 변화 시스템
                                                           │
                                                           └─► Launch Phase (Release)
                                                                │
                                                                ├─► 최종 밸런스
                                                                │
                                                                ├─► 최적화
                                                                │
                                                                └─► 스팀 출시
```

### 주요 의존성 상세

| 상위 작업 | 하위 의존성 | 설명 |
|-----------|------------|------|
| LAN 멀티 | 1인칭 컨트롤러 | 네트워크 동기화 전 로컬 조작 완료 필요 |
| 역할 선택 | 배 구조 확정 | 스폰 위치 설정을 위한 배 레이아웃 필요 |
| 네트워크 동기화 | 고래 AI 기초 | AI 결정을 호스트에서만 실행하도록 설계 |
| 체크포인트 | 페이즈 시스템 | 저장/로드할 데이터 구조 필요 |
| AI 고도화 | 네트워크 동기화 | 복잡한 동작도 동기화되어야 함 |
| 출시 준비 | 전 단계 완료 | 모든 기능 안정화 후 마케팅 자료 제작 |

## 리스크 매트릭스

### 기술적 리스크

| 리스크 | 확률 | 영향도 | 발생 단계 | 대응 전략 |
|--------|------|--------|-----------|-----------|
| Netcode 동기화 실패 | 중 | 상 | Alpha | Fallback: P2P 구조 또는 싱글플레이 집중 |
| Buoyancy 물리 과부하 | 중 | 중 | MVP | Unity Buoyancy 툴킷 교체 또는 단순화 |
| 고래 AI 복잡도 과다 | 중 | 중 | Beta | 패턴 단순화, LOD AI 적용 |
| 메모리 누수 | 중 | 상 | Launch | 프로파일러 지속 모니터링, 주간 테스트 |

### 일정 리스크

| 리스크 | 확률 | 영향도 | 발생 단계 | 대응 전략 |
|--------|------|--------|-----------|-----------|
| MVP 기능 과다 | 높 | 상 | MVP | Week 4에서 MVP 범위 재조정 가능 |
| 네트워크 디버깅 지연 | 중 | 중 | Alpha | Week 12까지 버그 허용, 이후 하향 조정 |
| 아트 작업 지연 | 중 | 중 | 전체 | 로우폴리 스타일로 리스크 최소화, 에셋 스토어 활용 |
| 스팀 심사 지연 | 낮 | 중 | Launch | Week 19에 미리 제출, 유동적 출시일 |

## 품질 게이트 (Quality Gates)

### Phase 완료 기준

#### MVP Gate (Month 2 말)
- [ ] 1인칭 시점에서 싱글플레이 가능
- [ ] 고래가 기본 패턴으로 공격
- [ ] AI 선장이 자동으로 배 조종
- [ ] 15분 세션 완료 가능
- [ ] **Go/No-Go 결정**: 멀티플레이어 구현 여부 확정

#### Alpha Gate (Month 3 말)
- [ ] LAN 4인 접속 및 플레이 가능
- [ ] 역할 선택 및 스폰 정상 작동
- [ ] 모든 역할의 상호작용 동기화
- [ ] **Go/No-Go 결정**: 온라인 매칭 추가 여부 확정

#### Beta Gate (Month 4 말)
- [ ] 고래 AI 복잡 패턴 구현
- [ ] 체크포인트 저장/로드 작동
- [ ] 페이즈별 환경 변화 적용
- [ ] **Go/No-Go 결정**: 출시 일정 확정

#### Launch Gate (Month 5 말)
- [ ] 버그 리포트 0건 (치명적)
- [ ] 스팀 페이지 승인
- [ ] 최종 빌드 테스트 완료
- [ ] **Release**

## 성공 기준 (Success Criteria)

### 기술적 성공
- 60fps 이상 유지 (GTX 1060 기준)
- LAN 환경 4인 동시 접속 안정성 99%
- 메모리 사용량 4GB 이하

### 게임플레이 성공
- 세션 완료율 80% 이상 (게임 오버 제외)
- 각 역할 재미 점수 7/10 이상 (내부 테스트)
- 1회 클리어 후 재플레이율 30% 이상

### 비즈니스 성공
- 스팀 출시 완료
- 사용자 평가 "복합적" 80% 이상
- 출시 첫 달 판매 1,000장 이상
