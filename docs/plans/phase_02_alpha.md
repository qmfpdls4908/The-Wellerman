# Phase 02: Alpha (LAN Multiplayer)
## 개발 기간: 1개월 (MVP 이후)

---

## 개요
MVP의 싱글플레이어 기반에 LAN 멀티플레이어 기능을 추가하여 2~4인 협동 게임플레이를 구현하는 단계입니다. Netcode for GameObjects를 활용한 로컬 네트워크 동기화와 역할 선택 시스템을 구축합니다.

---

## 목표
- Netcode for GameObjects 기반 LAN 멀티플레이어 구현
- 역할 선택 로비 시스템 (동적 2~4인 역할 배분)
- 4인 역할 구조 완성 (선장, 1등 선원, 포수×2)
- 네트워크 동기화 (캐릭터, 고래, 배 상태)
- 기본 음성 채팅 또는 프리셋 커뮤니케이션 시스템

---

## 구현 항목

### Week 1: 네트워크 인프라

#### Netcode 설정
- [ ] Netcode for GameObjects 패키지 설치
- [ ] NetworkManager 설정
- [ ] LAN 검색/연결 시스템 (로컬 네트워크 내 방 찾기)
- [ ] 호스트/클리언트 구조 설정
- [ ] 연결 안정성 체크 (Ping, 패킷 손실 감지)

#### 네트워크 오브젝트 설정
- [ ] 플레이어 캐릭터 NetworkObject화
- [ ] 고래 NetworkObject화
- [ ] 배 NetworkObject화 (부착된 플레이어와 함께 이동)
- [ ] 파이어 베이스 설정 (보류 - 향후 고려)

### Week 2: 역할 시스템 및 로비

#### 역할 선택 로비
- [ ] 로비 UI (방 만들기/참가하기)
- [ ] 역할 선택 UI (드래그 앤 드롭 또는 버튼 선택)
- [ ] 2~4인에 따른 동적 역할 활성화
  - 2인: 선장, 1등 선원
  - 3인: + 포수(좌현)
  - 4인: + 포수(우현)
- [ ] 역할 중복 방지 및 자동 배분
- [ ] 준비 완료 시스템 (모든 인원 Ready 시 게임 시작)

#### 역할별 스폰
- [ ] 선택한 역할에 따른 스폰 위치 설정
  - 선장: 선실 or 뒤편 갑판
  - 1등 선원: 갑판 중앙
  - 포수(좌): 좌현 포대
  - 포수(우): 우현 포대
- [ ] 역할별 초기 도구 지급
- [ ] 역할 교체 제한 (전사 시에만 다른 역할 겸임 가능)

### Week 3: 동기화 및 상호작용

#### 실시간 동기화
- [ ] 플레이어 위치/회전 동기화 (NetworkTransform)
- [ ] 플레이어 애니메이션 동기화
- [ ] 고래 위치/상태 동기화 (호스트 권한)
- [ ] 배 위치/회전 동기화
- [ ] 파도 상태 동기화 (시드 기반 동기화)

#### 상호작용 동기화
- [ ] 수리 진행도 동기화 (모든 플레이어가 같은 파손 상태 확인)
- [ ] 대포 발사 동기화 (발사 순간, 포탄 궤적)
- [ ] 보급품 회수 동기화
- [ ] 사기 게이지 동기화 (선장의 행동이 전원에게 반영)
- [ ] 화약 자원 동기화 (공유 자원)

#### 권한 관리
- [ ] 호스트 권한: 고래 AI, 페이즈 전환, 배 파괴 이벤트
- [ ] 클라이언트 권한: 자신 캐릭터 이동 (Client Authoritative)
- [ ] 서버 권한: 충돌 판정, 데미지 계산

### Week 4: 커뮤니케이션 및 테스트

#### 커뮤니케이션 시스템
- [ ] 프리셋 음성/텍스트 명령 시스템 (빠른 선택)
  - 예: "좌현에 고래!", "수리 필요!", "화약 보충!", "격노한다!"
- [ ] 위치 마커 시스템 (지정한 위치에 마커 표시)
- [ ] (Optional) 기본 음성 채팅 (WebRTC 기반)

#### 테스트 및 최적화
- [ ] 2인 LAN 테스트
- [ ] 4인 LAN 테스트
- [ ] 네트워크 지연 시뮬레이션 (인위적 Lag 테스트)
- [ ] 대역폭 최적화 (RPC 빈도 조절)
- [ ] 디스커넥션 처리 (재접속, AI 인계)

## 검증 기준 (완료 기준)

### 네트워크 기능
- [ ] 같은 LAN 내에서 방 생성/검색/입장 가능
- [ ] 4인 동시 접속 시 안정적 플레이 (10분 이상)
- [ ] 플레이어 간 위치/액션 실시간 동기화
- [ ] 호스트 이탈 시 게임 정상 종료 (또는 마이그레이션)
- [ ] 지연 100ms 이하 환경에서 플레이 가능

### 게임플레이
- [ ] 각 역할별 고유 위치에 정확히 스폰
- [ ] 좌/우 포수가 각자의 대포 독립적으로 조작
- [ ] 선장의 사기 회복이 모든 클라이언트에 실시간 반영
- [ ] 대포 발사가 모든 플레이어에게 보임
- [ ] 보급품 회수 시 모든 플레이어에게 사라짐

### UX
- [ ] 역할 선택 UI에서 직관적으로 역할 선택 가능
- [ ] 연결 끊김 시 명확한 메시지 표시
- [ ] 프리셋 명령으로 빠른 의사소통 가능

---

## 리스크 및 고려사항

### 기술적 리스크
- **네트워크 동기화 과부하**: 고래와 파도의 물리 동기화가 대역폭을 과다 사용
  - 대응: 호스트에서만 물리 계산, 위치/회전만 동기화 (5~10fps)
- **호스트 이탈**: 호스트가 나가면 게임 종료
  - 대응: 호스트 마이그레이션 구현 (복잡할 경우 우선순위 하향)

### 디자인 리스크
- **역할 불균형**: 특정 역할이 너무 한가하거나 바쁨
  - 대응: Week 3~4 낭심 테스트로 업묟 분배 조절

### 의존성
- MVP 단계의 1인칭 컨트롤러가 모듈화되어 있어야 함
- 고래 AI가 호스트에서만 실행되도록 설계되어야 함
