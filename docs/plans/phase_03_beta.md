# Phase 03: Beta (Advanced AI & Checkpoint)
## 개발 기간: 1개월 (Alpha 이후)

---

## 개요
고래 AI를 복잡한 행동 패턴으로 고도화하고, 페이즈 체크포인트 저장/로드 시스템을 구현하는 단계입니다. 게임의 핵심 재미인 "도전-학습-극복" 사이클을 완성하며, 난이도 조절과 반복 플레이를 위한 인프라를 구축합니다.

---

## 목표
- 고래 AI 복잡한 행동 패턴 구현 (격노, 잠수 반격, 분수분사)
- 페이즈 체크포인트 저장/로드 시스템
- 배 파괴 단계별 환경 변화
- 태풍 및 재난 시스템 간헐적 발생
- 난이도 설정 시스템 (캐주얼/노말/선원의 노래)

---

## 구현 항목

### Week 1: 고래 AI 고도화

#### 추가 행동 패턴
- [ ] **잠수 반격 상태**: 수면 아래로 완전히 잠수 후 배 밑에서 솟구침
  - 잠수 타이머 (3~5초)
  - 배 밑에서 올라오는 그림자 예고
  - 선원 "밑에서 온다!" 자동 외침 트리거
- [ ] **분수분사**: 등을 보이고 물뿌리기 (플레이어 시야 제한)
  - 1인칭 화면 물방울 효과
  - 3초간 시야 흐림
  - 조타수(선장)만 의존해야 하는 상황 연출
- [ ] **격노 모드**: 체상 30% 이하 시 진입
  - 이동 속도 1.5배 증가
  - 공격 쿨다운 50% 감소
  - 눈빛/색상 변화 (빨간색 테두리)
- [ ] **회피 패턴**: 포탄 예측하여 회피 (30% 확률)

#### AI 감각 시스템
- [ ] 청각: 대포 발사 소리 위치 추적
- [ ] 시각: 배의 움직임 감지
- [ ] 거리 기반 행동 변화 (근접/중거리/원거리)

### Week 2: 페이즈 체크포인트 시스템

#### 저장 시스템
- [ ] 페이즈별 데이터 스키마 설계
  - 고래 현재 체상
  - 현재 페이즈 번호
  - 배 파괴 상태 (어떤 부분이 부서졌는지)
  - 보유 도구/자원 목록
  - 사기 현재 수치
- [ ] JSON 기반 세이브 파일 구조
- [ ] 저장 시점 트리거 (페이즈 전환 시 자동)
- [ ] 수동 저장 (설정 메뉴에서)

#### 로드 시스템
- [ ] 저장된 페이즈부터 시작 옵션
- [ ] 상태 복원 로직
  - 고래 체상 복원
  - 배 상태 복원 (파괴 부위 유지)
  - 사기 50%로 회복 (중간값)
  - 보급품 기본 제공량으로 리셋
- [ ] 네트워크 환경에서의 동기화 (모든 플레이어 동일 상태)

#### 게임 오버/재시작 UI
- [ ] 게임 오버 화면 (사기 0% / 배 전파 / 보급품 고갈)
- [ ] 재시작 선택지 UI
  - 처음부터 시작
  - 페이즈 2 (75%)부터 시작 [잠김/해금]
  - 페이즈 3 (50%)부터 시작 [잠김/해금]
  - 페이즈 4 (25%)부터 시작 [잠김/해금]
- [ ] 페이즈 해금 조건 (해당 페이즈 도달 시 자동)

### Week 3: 환경 및 재난 시스템

#### 배 파괴 단계
- [ ] **1단계 (100%)**: 완전한 배
- [ ] **2단계 (75%)**: 좌현/우현 중 하나 파괴
  - 해당 측면 대포 사용 불가
  - 갑판에 구멍 생김 (낙하 위험)
- [ ] **3단계 (50%)**: 갑판 중앙 파괴
  - 이동 경로 제한
  - 수리 지점 추가
- [ ] **4단계 (25%)**: 마스트 파괴
  - 시야 제한
  - 최종 결전 분위기 연출

#### 재난 시스템
- [ ] **태풍**: 2~3분 간격으로 발생
  - 배 전체 흔들림 증가
  - 사기 감소 속도 2배
  - 고래 공격력 증가
  - 시각적 효과 (어두운 구름, 번개)
- [ ] **거센 파도**: 태풍 중 간헐적 발생
  - 플레이어 넘어짐 (3초간 행동 불가)
  - 보급품 바다로 날아감
  - 버튼 연타로 균형 잡기 (또는 자동 회복)
- [ ] **물고기 떼**: 보급품 대신 회수 가능한 소소한 자원

#### 난이도 설정
- [ ] **캐주얼**: 파도 약함, 고래 둔함, 사기 감소 느림, 체크포인트 자동 저장
- [ ] **노말**: 기본 설정
- [ ] **선원의 노래**: 파도 강함, 고래 예측 불가, 사기 감소 빠름, 체크포인트 없음 (1회 클리어 후 해금)

### Week 4: 사운드 및 폴리싱

#### 사운드 디자인
- [ ] BGM: "The Wellerman" 어레인지 (메뉴, 전투, 페이즈별 변주)
- [ ] 환경음: 파도, 바람, 우레
- [ ] 효과음: 대포, 고래 울음, 수리 소리, 외침
- [ ] 사기 관련 음악: 선장 연주 시 악기 소리 (밴조)

#### UI/UX 개선
- [ ] 1인칭 손/도구 애니메이션 세부화
- [ ] 데미지 받을 때 화면 테두리 효과
- [ ] 고래 위치 인디케이터 (수면 아래일 때)
- [ ] 미니맵 (선장 전용)
- [ ] 튜토리얼 팝업 (첫 플레이 시)

#### 밸런스 조정
- [ ] 각 페이즈 플레이 시간 측정 (목표: 4~5분/페이즈)
- [ ] 화약 소모량 vs 고래 체상 감소율 조정
- [ ] 사기 감소/회복 속도 조정

## 검증 기준 (완료 기준)

### AI
- [ ] 고래가 30% 이하에서 격노 모드 진입
- [ ] 잠수 후 배 밑에서 공격 시 사전 경고 있음
- [ ] 분수분사 시 플레이어 시야 제한됨
- [ ] 고래가 때로는 포탄을 회피함

### 체크포인트
- [ ] 75% 도달 후 게임 종료 시 해당 지점부터 재시작 가능
- [ ] 저장된 상태 정확히 복원 (배 파괴 상태 유지)
- [ ] 네트워크 환경에서도 모든 플레이어 동일 상태로 로드
- [ ] 1회 클리어 전에는 직전 체크포인트만 선택 가능

### 환경
- [ ] 페이즈별로 배 외형 변화 (구멍, 파손)
- [ ] 태풍 발생 시 시각적/청각적 변화 명확
- [ ] 거센 파도에 플레이어 영향받음 (넘어짐)
- [ ] 난이도 변경 시 체감 가능한 변화 있음

### 품질
- [ ] BGM이 페이즈/상황에 따라 자연스럽게 전환
- [ ] 1인칭 애니메이션 자연스러움
- [ ] 20분 세션 낭심 플레이 테스트 완료

## 리스크 및 고려사항

### 기술적 리스크
- **세이브 데이터 호환성**: Alpha에서 Beta로 넘어가며 구조 변경 시 기존 세이브 깨짐
  - 대응: 버전 넘버링, 불일치 시 초기화 처리
- **복잡 AI 성능**: Behavior Tree가 과도한 CPU 사용
  - 대응: LOD AI (가까울 때만 복잡한 계산)

### 디자인 리스크
- **페이즈 난이도 급격**: 75%에서 50%로 넘어갈 때 난이도 상승이 너무 급격
  - 대응: Week 4 밸런스 테스트로 조절

### 의존성
- Alpha 단계의 네트워크 동기화가 안정적이어야 함
- MVP의 페이즈 시스템 기초가 모듈화되어 있어야 함
